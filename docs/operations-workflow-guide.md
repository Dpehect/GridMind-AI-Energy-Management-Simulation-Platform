# Operations Workflow Guide

## Work orders

1. Create and assign.
2. Start work.
3. Add comments and evidence.
4. Consume inventory parts.
5. Complete checklist.
6. Record actual cost and outcome.
7. Close and audit.

## Inventory

Every stock change is represented by an immutable movement record.

## Purchase requests

Requests begin as draft, then require workspace-admin approval.

## Goals

Progress entries preserve historical values while the goal record stores the current value and derived status.

## Reports

Reports move from draft to review, then approved, rejected or changes requested.

## Recurring maintenance

Plans define cadence and next execution time. Job generation will be automated in the background-jobs phase.
